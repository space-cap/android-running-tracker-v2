# Running Tracker V2 - 남은 작업 목록 (Remaining Tasks)

현재 프로젝트의 핵심 기능(위치 추적, 서비스, 기본 UI)은 구현되었으나, 앱의 완성도를 높이기 위해 필요한 남은 작업들을 정리합니다.

## 1. 데이터 영속성 (Persistence)
- [ ] **러닝 기록 저장 로직 구현**: `RunningService` 또는 `HomeScreen`에서 러닝 종료 시 `TrackingManager`의 데이터를 기반으로 `RunRecord`를 생성하고 Room DB에 저장하는 기능을 추가해야 합니다.
- [ ] **지도 스냅샷 캡처**: `RunRecord` 엔티티에 정의된 `img` 필드를 활용하기 위해, 러닝 종료 시점의 지도 화면을 Bitmap으로 캡처하여 함께 저장하는 기능이 필요합니다.

## 2. 화면 추가 및 내비게이션 (Navigation)
- [ ] **내비게이션 구조 설계**: `MainActivity`에 `NavHost`를 도입하여 여러 화면 간 이동을 관리해야 합니다.
- [ ] **러닝 기록 목록 화면 (Run List Screen)**: 저장된 모든 러닝 기록을 리스트 형태로 보여주는 화면입니다. 정렬 기능(날짜, 거리, 시간 등)이 포함되어야 합니다.
- [ ] **기록 상세 화면 (Run Detail Screen)**: 특정 러닝 기록을 클릭했을 때 지도 경로와 상세 통계(평균 페이스, 총 거리 등)를 보여주는 화면입니다.
- [ ] **통계 화면 (Statistics Screen)**: 주간/월간 단위의 총 거리, 평균 페이스 등을 그래프나 요약 정보로 보여주는 화면입니다.

## 3. UI/UX 개선 (Refinement)
- [ ] **카메라 추적 모드**: 러닝 중 사용자의 현재 위치가 지도의 중앙에 오도록 카메라를 자동으로 이동시키는 기능을 추가해야 합니다.
- [ ] **백그라운드 위치 권한 처리**: Android 10 이상에서 안정적인 추적을 위해 '항상 허용(Allow all the time)' 권한 요청 플로우를 보완해야 합니다.
- [ ] **중지/재개 기능**: 현재는 시작과 종료만 가능합니다. 일시 중지 및 재개 기능을 추가하여 사용자 편의성을 높여야 합니다.

## 4. 테스트 및 품질 관리 (Quality)
- [ ] **단위 테스트 완료**: `RunDaoTest` 등 데이터 계층의 테스트를 마무리하고, `TrackingManager`의 로직 테스트를 추가해야 합니다.
- [ ] **UI 테스트**: Compose Test를 통해 지도 인터페이스와 버튼 동작을 검증해야 합니다.
- [ ] **리소스 최적화**: 배터리 소모량 확인 및 포그라운드 서비스의 알림 디자인 개선이 필요합니다.
